Chapter 01 — The Beginning
01
An engineer who always asked — "but what if it breaks?"

Growing up in Maharashtra, I was the kid who needed to understand not just how things worked — but how they failed. That instinct led me straight to QA. After my Bachelor of Engineering I joined Maxso Technologies, testing UBS Neo — a global trading platform for one of the world's largest investment banks. The stakes were real from day one.

🏛️ Capital Markets 🐛 Manual Testing 📋 Test Strategy 🏦 UBS Neo
Testing is not about finding bugs. It's about understanding a system so deeply that you know where it will fail before it does.
Chapter 02 — Growth
02
From trading floors to telecom towers — falling in love with automation

After capital markets I moved into telecom — testing OSS/BSS billing and CRM systems for Datagate USA. Validating CDRs for thousands of subscribers taught me that data integrity at scale is everything, and automation is how you protect it. I invested in Java, TestNG, Maven, and the Page Object Model, writing Selenium scripts that ran overnight and delivered clean reports by morning.

🤖 Selenium WebDriver ☕ Java 🧪 TestNG 📦 Maven POM 📱 Telecom OSS/BSS

FinTech, real money, zero margin for error

Every bug has direct financial consequences. I built an automation framework that reduced regression effort by 60% and I'm still the engineer who asks "what if it breaks?"

Why I Chose QA

Not everyone can break things for a living.

Most people in tech want to build. I wanted to understand systems so completely that I could predict every way they'd fail. QA was a deliberate choice.

In an industry obsessed with shipping fast, QA engineers ask the uncomfortable questions. What does this do with unexpected input? What happens under load? Does this compliance flow actually match the regulation?

I chose QA because software quality is a form of respect — for users, for business, and for the craft of engineering. Every bug I catch is a problem a real user will never face.

"ANYONE CAN SHIP. NOT EVERYONE CAN SHIP AND SLEEP AT NIGHT."

My Process

How I approach every project.

01📋
Requirement Analysis ↗
Study FRS, BRS, user stories. Identify gaps and testing risks before any code is written.
02📝
Test Planning ↗
Design test strategy, estimate effort, define scope, set up environments and test data.
03⚗️
Test Design ↗
Write precise test cases for functional, edge, negative, integration and regression scenarios.
04🚀
Execute & Automate ↗
Manual first, then automate stable flows. Report all defects in JIRA with clear reproduction steps.
05✅
Sign-off & Closure ↗
Verify critical defects resolved, update reports, provide go/no-go recommendation for release.
